# 1. Definição do MVP
Meu MVP foca na **"Operação Essencial de Loja e Retaguarda Básica"**. 

- **O que está dentro do MVP:** O fluxo completo de balcão (PDV) que vai da busca do produto à emissão do comprovante, englobando o controle imediato de estoque, validação de receitas, vendas a prazo, cadastro rápido de clientes, entrada de compras de fornecedores e o controle financeiro básico da unidade (contas a pagar e receber).
- **O que está fora do MVP:** Integração fiscal complexa (emissão de NF-e completa, SPED), consolidação em tempo real de dados na matriz envolvendo dezenas de filiais simultâneas, e o módulo de Recursos Humanos/Folha de Pagamento.
- **Por que fiz essas escolhas:** Para garantir que uma unidade da farmácia consiga operar de forma independente, ágil e sem furos de estoque/caixa no dia a dia. Com essa base sólida (MVP), a integração corporativa mais robusta pode ser implementada em uma segunda fase.

# 2. Regras de Negócio (mínimo: 5)
**RN01 — Venda Restrita por Estoque:** Um produto não pode ter sua venda finalizada se a quantidade solicitada for maior que a quantidade física disponível no estoque da unidade.  
**RN02 — Atualização Imediata de Estoque:** O saldo de estoque deve ser atualizado automaticamente e em tempo real após vendas, devoluções ou entradas de compras.  
**RN03 — Venda a Prazo Exclusiva:** O parcelamento só é permitido para clientes cadastrados no sistema, gerando automaticamente um título no "Contas a Receber".  
**RN04 — Validação de Receita:** A venda de medicamentos controlados só pode ser liberada no sistema mediante aprovação de um usuário com perfil "Farmacêutico".  
**RN05 — Alerta de Nível Mínimo:** O sistema deve alertar a gerência automaticamente quando o estoque de um produto atingir o nível mínimo configurado.

# 3. Requisitos Funcionais (mínimo: 8)
**RF01 —** O sistema deve permitir o registro, edição e consulta de clientes.  
**RF02 —** O sistema deve permitir que o Atendente registre vendas pesquisando itens por nome, código de barras ou fabricante.  
**RF03 —** O sistema deve verificar a disponibilidade e dar baixa automática de itens no estoque durante a venda.  
**RF04 —** O sistema deve permitir o lançamento de compras de fornecedores, alimentando o estoque da unidade.  
**RF05 —** O sistema deve gerar lançamentos no módulo "Contas a Pagar" ao confirmar o recebimento de uma compra.  
**RF06 —** O sistema deve gerenciar "Contas a Receber", atualizando o status de vendas a prazo (Aberta, Recebida, Atrasada).  
**RF07 —** O sistema deve emitir um comprovante detalhado ao final de cada venda.  
**RF08 —** O sistema deve gerar relatórios operacionais (produtos mais vendidos, estoque atual, vendas por período).

---

# 🛡 4. Requisitos Não Funcionais (mínimo: 4)
**RNF01 — Segurança:** Acesso restrito por perfis de usuário (Atendente, Farmacêutico, Gerente, Financeiro).  
**RNF02 — Usabilidade:** A interface do PDV deve suportar atalhos de teclado para acelerar o atendimento.  
**RNF03 — Desempenho:** Consultas de produtos no banco de dados devem retornar em no máximo 1 segundo.  
**RNF04 — Disponibilidade:** O sistema deve ter um *uptime* de 99,9% durante o horário de funcionamento das lojas.

# 6. Documentação dos Casos de Uso

## **UC01 — Registrar Venda**
**Ator(es):** Atendente  
**Descrição:** Processo central onde o atendente registra os produtos desejados pelo cliente, recebe o pagamento e finaliza a compra.  
**Pré-condições:** Usuário autenticado e caixa aberto.  
**Pós-condições:** Venda concluída, estoque atualizado e caixa atualizado.  

### Fluxo Principal
1. Atendente inicia uma nova venda.  
2. Atendente inclui produtos na venda (Include UC02 e UC03).  
3. Sistema calcula o total da compra.  
4. Atendente seleciona a forma de pagamento (Dinheiro/Cartão).  
5. Sistema processa o pagamento e emite recibo (Include UC04).  

### Fluxos Alternativos / Exceções
- **FA01 — Medicamento Controlado:** Se o produto exigir receita, o sistema pausa a venda e aciona o fluxo de validação (Extend UC07).  
- **FA02 — Venda a Prazo:** Se o cliente desejar pagar a prazo, o sistema direciona para validação de crédito (Extend UC06).  
- **FA03 — Cliente não cadastrado:** O atendente pode pausar para registrar o cliente (Extend UC05).

### Relacionamentos
- **Include:** UC02, UC03, UC04  
- **Extend:** UC05, UC06, UC07

## **UC09 — Gerenciar Contas**
**Ator(es):** Financeiro  
**Descrição:** Controle de entradas (convênios/prazo) e saídas (fornecedores/despesas).  
**Pré-condições:** Usuário com permissão financeira logado.  
**Pós-condições:** Status de títulos financeiros atualizados.  

### Fluxo Principal
1. Financeiro acessa o painel de Contas.  
2. Filtra títulos por vencimento.  
3. Seleciona um título "Aberto".  
4. Registra a data de liquidação e altera o status para "Pago" ou "Recebido".  

### Fluxos Alternativos / Exceções
- **FA01 — Pagamento Parcial:** Financeiro altera o valor recebido; sistema mantém o título "Aberto" com o saldo residual.
